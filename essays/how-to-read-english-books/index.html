本文重点讲的是 how , 而不是 why, 关于 why 的文章网上已经很多了, 这里说一下 how.

首先你英语要过关, 所谓过关, 大约英语四级的水平就够了, 技术类书籍的词汇量真心不大, 就是一些专业术语, 看过几本书就大概知道了八九分了, 重点是概念的理解.

先说说我自己走过的弯路吧：

首先就是用看小说的方式来看技术书籍, 一股脑的从头看到尾, 很容易造成看悬疑小说一样的情况, 迷雾重重, 其实应该先看目录和前言. 知道这本书大概讲了什么, 现在是网络时代, 书本固然有书本的好处, 但是现在信息的来源很多样化, 完全可以自己通过现有的网络内容”预习”一下书本知识. 比如你看一本 < xxx in action > ,你总得上 xxx 的官网看一看 getting started 吧. 看的过程中你肯定会有一些疑惑, 带着疑惑来看书效果更好.

还有就是不要无谓的为了看而看, 这不是说工作中用不上的相关知识就不要学, 而是说看一本书一定要把书中的内容用起来. 比如你看一本网页制作相关的书, 就应该自己制作一个网站, 把书中用到的知识都用起来, 这样也能起到加强记忆的功能.

上面说到的两点和 英文 关系不大, 但是由于英文毕竟不是母语, 阅读起来比中文要困难, 犯了上述错误的损失比较大, 所以特地提一下.

对于已经学会的技术, 如果有新的书出来, 我还是会翻一翻的, 看这类书就比较轻松了, 因为大局观已经有了, 就可以跳着看了, 挑自己感兴趣的部分, 看看作者对这个部分是一个什么思路, 或者看看自己没用过的一些高级特性. 比如< Crafting Rails 4 Applications >, 在我刚学 rails 的时候, 看上去完全是天书. 现在能读出味道了. 还有< The Linux Command Line >, 我认为这本书适合已经具备基本的 Linux 操作的人看, 非常系统完整, 语言平实, 但是全书没有一句废话, 每一章都是精华, 看过之后犹如醍醐灌顶, 强烈推荐.

下面推荐一些出版商和系列丛书 推荐的出版社和系列(manning in action pub pack cookbook)

  • The Pragmatic Programmers
  • Manning
  • O'Reilly
  • Packtpub

最后说一下看英文书要克服的障碍 + 坚持, 这是必须的, + 一章一章看, 写博客笔记, 看完一章要回顾一遍

按照这个节奏, 只要坚持看完两三本, 跨过这条坎, 那以后的路就轻松了,

感谢 Ruby 和 Rails, 如果我继续在搞 Java, 那么有大量现成的中文材料可以看, 也许我就不会去看英文书了... 反过来说, 要接触最新最潮的技术, 那么看英文材料是必须的.

差点忘了, 如果要买国外正版电子书的话我推荐leanpub
