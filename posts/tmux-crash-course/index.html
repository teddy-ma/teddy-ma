tmux 快速入门

为什么要用 tmux

如果你对 tmux 的疑问还停留在：我可以开多个 terminal 的 tab 啊， 为什么要用 tmux 呢？那我只能呵呵了。 tmux 的强大之处在于：

  • 可以连接本地或远程的 sessions
  • 强大的 window 和 pane 管理
  • 在不同的 session 之间移动 window
  • 自动化脚本

基本概念

tmux 这个单词是 terminal multiplexer 的缩写。 tmux 可以让你在一个屏幕中管理多个终端窗口。 先看图：

session

tmux 是被设计为 cs 结构的。因此当你启动 tmux 的时候, 实际上你启动的是 tmux 的 client。 服务端可能也在你的电脑上或者其他服务器上，总之要连接 client 和 server， 就需要一个 session。一般来说一个 session 对应这一个 project。

window

一个 session 包含多个 window，这里的 window 比较类似一般 terminal 模拟器的 tab。

pane

一个 window 可以被分割为多个 pane，方便同时查看多个进程的状态。

快捷键

tmux 的快捷键可以非常灵活地自定义

配置

tmux 的配置文件位于 $HOME/.tmux.conf (以及一个全局的 /etc/tmux.conf) 和大多数 dotfile 一样，把配置项写到这个文件中，启动 tmux 的时候就会自动读取了。 下面是一些常用配置。

前缀

这个是所有快捷键的基础，设定了一个 prefix，系统才能知道你后续的操作是对于 tmux 的。 几乎所有的快捷键都是要先输入 prefix 的。默认的 prefix 是 C-b 。不过这和 emacs 的 back 冲突了。 网上比较主流的设定是 C-a ，一来这两个键很近，二来虽然这和 emacs 的回到行首冲突，不过相对于 C-b, C-a 多按几次也没有副作用，并且用的频率没那么高，所以大多数人都会选择用 C-a 作为 prefix。 当然，也有人觉得 C-t （t for tmux）是一个不错的选择，anyway，把 prefix 当成一个变量即可。

配置项：

set -g prefix C-a # 设定前缀
unbind C-b  # 取消默认的前缀
set -s escape-time 1 # 设定前缀的作用时间
bind C-a send-prefix # 确保可以在其他应用中可以发送 Ctrl-A

常用操作

这里列出的快捷键都是按照上面的配置配好之后的结果。 其中有些操纵是默认值所以并没出现在配置 list 中。

使用 prefix - ? 可以随时查看快捷键列表。

session

配置项：

bind C-c new-session # 创建新的 session
bind C-f command-prompt -p find-session 'switch-client -t %%' # 查找 session

说明：

功能 快捷键
新建 session prefix - C-c
返回 session tmux attach -t session-name
列出 sessions tmux list-sessions
查找 session prefix - C-f
重命名 session tmux rename-seesion session-name
切换 session tmux switch -t session-name

window

配置项：

bind -r C-h previous-window # 选定上一个 window
bind -r C-l next-window     # 选定下一个 window
功能 快捷键
新建 window prefix-c
关闭 window prefix-&
重命名 window prefix-,
列出 windows prefix-w
切换 window prefix-[0-9]

panel

配置项：

bind - splitw -v # 分割成上下两个窗口
bind | splitw -h # 分割成左右两个窗口
功能 快捷键
横向切分 window prefix - 竖杠
纵向切分 window prefix - 横杠
最大化 pane prefix-z
改变 pane 的 layout prefix-space
调整 pane 尺寸 prefix 然后按住(shift)FBNP
切换 pane(顺序跳转） prefix - o
切换 pane(选择编号) prefix - q 然后按对应的编号

模式

正如 vi 有不同的模式一样，tmux 也有自己的模式：

default mode

默认模式，就是我们一直在操作的模式。

copy mode

快捷键 prefix-[ 就会进入 copy mode。在这个模式下，你可以查看历史记录， 复制粘贴内容，类似于 vi 的 normal mode。

command mode

类似于在 emacs 下直接输入指令。一些复杂或冷门的操作可以在这里直接调用。

clock mode

快捷键 prefix-t, 会显示一个时钟。。。这真的不是一个彩蛋吗。。。 想要把 prefix 设置成 t 的同学不需要担心覆盖掉这个模式了，这模式压根就没用啊。。。

结对编程

如果在结对编程时两个脑袋凑在一个屏幕前是不是有点挤啊。。。 有了 tmux 就不怕啦。只需要一个 server 端，然后两个 client 一起连上去。 不但可以共享『屏幕』，还能一起操作同一个文件。 这样的 common pattern 已经有人封装好了工具了，就是这个 -- wemux。 不但可以选择共享模式（mirror,pair,rogue）。还可以踢人哦。。。
